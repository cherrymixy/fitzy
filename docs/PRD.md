# FITZY — MVP PRD

> 매일 코인 한 개로 그날의 **추구미(fit·무드)** 를 뽑고, 그 무드대로 하루를 살며 **9칸 보드**를 이미지로 채워,
> 추구미를 **'도달 가능미'** 로 만들어가는 데일리 무드 기록 앱. Flutter + Dart.
> 태그라인: **"추구미를 도달 가능미로."**

---

## 0. 작업 방식 (BLACK MAGIC 스타일)

1. **로직 먼저 수렴.** 상태(코인/무드/보드/프로필)·데이터·핵심 규칙을 UI와 분리해 *먼저 동결*. 콘텐츠(무드·카테고리)는 `data/`로 외부화.
2. **디자인은 두 층.** (a) 토큰·셸은 초반에, (b) 화면별 비주얼·연출은 로직 동결 후 마지막 패스.
3. **빌드는 복붙 단계별 프롬프트로** 쪼갠다. 규칙은 `CLAUDE.md`에 영속화.
4. **한 번에 하나 → 결과 확인 → 다음.** 묶지 않는다.
5. **시각 조정은 모호한 말 금지.** 판단은 사람이, 도구엔 정확한 스펙으로. 측정 스크립트 금지.
6. **핸드오프 정밀도 사다리:** 말 → 값 → 이미지 → **Codex가 만든 HTML/CSS(구조화 비주얼 스펙)**. 막히면 한 단계 올린다.
7. **비주얼은 Codex가, 로직은 Claude가.** Codex가 화면별 HTML/CSS를 만들면 Claude가 그걸 **theme 토큰 기반 Flutter 위젯으로 '번역'(로직 불변)**.

## 1. 개요
- **한 줄 정의:** 코인으로 그날의 무드를 뽑고, 그 무드대로 9칸 보드를 이미지로 채워 캘린더에 누적하는 데일리 무드 앱.
- **핵심 루프:** 뽑기(Select) → 채우기(Board) → 기록(Calendar).
- **왜:** '되고 싶은 미감(추구미)'을 매일 가볍게 뽑고, 실제 하루의 사진으로 채워 *도달 가능한 것*으로 바꾼다.

---

## 2. 목표 & 비목표 (MVP)

**포함**
- 회원가입/로그인 **(최소)**, 온보딩 4스텝.
- 일일 코인 1개(자정 리셋).
- **Select**: 코인 넣고 9무드 중 뽑기(코인 소모, 하루 1회).
- **Board**: 뽑은 무드의 9칸을 촬영/갤러리 이미지로 채우기. 부분 저장·자정 마감.
- **Calendar**: 완성 보드만 노출 + 월 통계.
- **My**: 프로필 이미지/닉네임/아이디/성별(비공개 가능)/태그(선택) + 설정·공지·고객센터.
- 상단 코인 표시, 하단 4탭.

**비목표(부록 D)**
- 콜라주 공유/내보내기, 이미지 편집(크롭/필터), 추가 코인·리워드, 푸시 알림, 소셜/팔로우·댓글, 다국어, 클라우드 백업(경로 A 기준).

## 5. 화면 정의

공통: 상단 **FITZY 워드마크 + 코인 칩(S 1)**, 하단 **4탭(Select / Board / Calendar / My)**.

### 5.1 Auth (회원가입/로그인) — 최소
- 입력: 닉네임, 아이디(**내부 유니크**), 성별(선택·**비공개 가능**), 태그(선택). 프로필 이미지(선택).
- 로그인은 최소 구성. 상세는 **§10 계정 전략**에서 경로 택1.

### 5.2 Onboarding (4스텝)
- 위 4줄 카피를 슬라이드로. 마지막에 시작 CTA.

### 5.3 Select (Pick Your Fit)
- 3×3 무드 그리드(J1~J9), 우측 자판기 패널(INSERT COIN / RETURN / 키패드 — 비주얼).
- 코인 있으면 무드 하나 **픽** → 코인 1 소모 → 오늘 무드 확정 → Board로.
- 코인 없으면(이미 뽑음/미지급) "내일 다시" 상태.

### 5.4 Board
- 상단: **제목 = 뽑은 무드명(편집 가능, 연필)**, 날짜, **N images**(채운 칸 수), 즐겨찾기 하트.
- 3×3 그리드: **Food / Outfit / Color · Hobby / Place / Activity · Music / Work / Drink**.
- 빈 칸 탭 → 바텀시트(카메라/갤러리) → 이미지 + 즉시 저장. 채운 칸 탭 → 교체/삭제. 하단 "이미지 추가".
- 자정 지나면 마감(수정 불가).

### 5.5 Calendar
- 월 그리드 + 좌우 월 네비. **완성 보드만** 해당 날짜에 썸네일/표식.
- 하단 통계: **이번 달 기록 N일**, **가장 많이 기록한 무드**.

### 5.6 My
- 프로필 이미지(편집), 닉네임(편집), @아이디 | 성별, 태그(#무드들, 선택).
- 리스트: 설정 / 공지사항 / 고객센터. 그 외는 최소.

## 7. 코인 · 뽑기 · 보드 규칙 (*동결 대상*)

1. **코인:** 앱 진입 시 오늘 `dateKey` ≠ `lastGrantedDateKey`면 1개 지급, `drawnToday=false`. 같은 날 추가 지급 X. 이월 없음.
2. **뽑기(Select):** 코인 있으면 9무드 중 하나 픽 → 코인 1 소모, `drawnToday=true`, 오늘 `DayRecord`(무드 확정) 생성. 오늘 재뽑기 X, 내일 가능.
3. **채우기(Board):** 확정된 오늘 무드의 9칸을 이미지로(촬영/갤러리). 칸당 **1장**, 채울 때마다 **즉시 저장**, 교체/삭제 가능. 제목(무드명) 편집·즐겨찾기 토글.
4. **마감:** 자정(기기 로컬) 경계 넘으면 그날 보드 finalize(수정 불가).
5. **완성 & 캘린더:** 완성 = 9/9. **캘린더엔 완성된 보드만 노출**(부록 D에서 '부분도 흐리게' 옵션 검토).
6. **테스트:** `now`(날짜) 주입 가능 → 경계 케이스 단위테스트.

> 가정(확인 필요): Select는 **"Pick Your Fit"** 이라 사용자가 9개 중 직접 고르는 방식으로 봄(랜덤 가챠 아님).

## 9. 이미지 처리
- **입력:** `image_picker` — 카메라 + 갤러리.
- **영구화:** picker 임시 파일을 `path_provider` documents 하위로 **복사** 후 경로 저장. 교체/삭제 시 옛 파일 정리.
- **권한(iOS):** `NSCameraUsageDescription`, `NSPhotoLibraryUsageDescription`(한국어). 거부 시 안내+설정 유도.
- **저작권:** 개인 로컬 아카이브 OK. 향후 콜라주 외부 공개 기능 시 타인 저작 이미지 공개 주의(MVP는 공유 비포함).

## 10. 계정/데이터 전략 (🟡 핵심 결정 — 택1)

| | 경로 A — 로컬 우선 (권장·MVP) | 경로 B — Firebase |
|---|---|---|
| 회원가입/로그인 | 온보딩서 닉네임·아이디 설정(명목 로그인) | Auth(이메일/소셜) 실제 계정 |
| 아이디 유니크 | 로컬 형식검증(전역 보장 X) | Firestore로 전역 유니크 보장 |
| 데이터 | SharedPreferences + 로컬 파일 | Firestore + Storage |
| 멀티기기 | X | 동기화 O |
| 비용 | 빠름·단순 | async/보안규칙/에러상태 ↑ |

> 데이터 계층을 **Repository로 추상화**해 A↔B 갈아끼우게 설계. "그외 최소한" 기조상 MVP는 **경로 A 권장**, 진짜 계정/유니크가 필수면 B.
